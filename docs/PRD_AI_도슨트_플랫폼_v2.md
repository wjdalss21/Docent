## 1. 프로젝트 개요

### 배경 (논문 연구 배경 기반)
국립중앙박물관 연간 관람객 400만 명 이상으로 미술관·박물관의 대중화가 가속되고 있으나, 기존 도슨트 서비스는 세 가지 구조적 한계를 가진다.

- **주목 지점 부재**: 처음 작품을 마주한 관람객이 "어떤 관점으로 어디에 주목해야 할지" 안내 부족
- **일방향 정보 전달**: 오디오 중심 해설은 오히려 인지 부하를 높여 몰입 방해
- **개인화 부재**: 동일한 설명으로 관람객의 특정 관심 속성에 맞춘 해설 제공 불가

본 서비스는 BERTopic 모델링으로 도출한 4가지 도슨트 경험 속성(작품 배경 / 작품 의미·상징 / 작품 관계성 / 전달 방식)을 기반으로, 관람객이 원하는 관점과 톤을 선택하면 Gemini AI가 개인화된 해설을 생성하는 모바일 퍼스트 웹 서비스다.

### 연구 기반 속성 구조 (BERTopic 결과)

| 속성 | 분류 | 역할 | 대응 UX 요소 (Honeycomb) |
|------|------|------|--------------------------|
| 작품 배경 | 설명적 속성 | "이런 작품이 만들어진 배경은 무엇인가" | 유용성, 신뢰성 |
| 작품 의미·상징 | 설명적 속성 | "이 작품이 무엇을 말하고자 하는가" | 매력성, 가치성 |
| 작품 관계성 | 연결적 속성 | "이 작품은 어떤 작품과 유사한 특징을 가지는가" | 탐색성, 가치성 |
| 전달 방식 (톤) | 표현적 속성 | "이 내용을 관람객에게 어떻게 전달할 것인가" | 매력성, 사용성 |

### 데이터 소스
- **서울시립미술관 작품 데이터** 활용 (공공데이터 포털 or 직접 구축)
- 작품 메타 정보: 제목, 작가, 연도, 기법, 재료, 크기

### 목표
- 논문의 프로토타입을 **실제 동작하는 서비스**로 구현
- 포트폴리오 + 졸업작품 제출용 완성도 확보
- 향후 연구방향(Self-RAG, Langchain)으로 확장 가능한 구조 설계

## 2. 사용자 플로우

```
[홈 - 작품 목록]
    ↓ 작품 카드 클릭
[작품 상세]
    → "원하시는 설명 속성을 선택해주세요" (초기 상태)
    ↓ 속성 탭 선택 (작품 배경 / 작품 의미·상징 / 작품 관계성)
[AI 해설 생성 & 스트리밍 출력]
    ↓ 해설 하단 "톤앤매너 변경" 버튼
[톤앤매너 선택 화면]
    → 정식적·해설형 / 유머러스·친근형 / 어린이 맞춤형 / 철학적·사색형
    ↓ 선택 시 해당 속성 + 톤으로 재생성
[추가 질문 채팅]
    → 메시지 입력 → 멀티턴 대화
```

**톤 선택 타이밍 결정**: 속성 선택 후 → 해설 생성 → 하단에서 톤 변경 가능 방식
(이유: 먼저 기본 해설을 보고 나서 톤을 바꿔보는 게 더 자연스러운 탐색 경험)

## 3. 핵심 기능 명세

### F1. 작품 목록 (홈)
- 서울시립미술관 작품 카드 그리드 (썸네일 + 작품명 + 작가명)
- 상단 "검색🔍" — 작품명 / 작가명 실시간 필터
- 각 카드에 "AI 도슨트" 뱃지 표시
- TanStack Query로 페이지네이션 or 무한 스크롤

### F2. 작품 상세
- 작품 이미지 (고화질, Supabase Storage)
- 메타 정보: 방법 / 재료 / 스탠다드(크기)
- 초기 상태: "원하시는 설명 속성을 선택해주세요" 안내
- 하단 고정 탭: [작품 배경] [작품 의미·상징] [작품 관계성]

### F3. AI 해설 생성 (핵심)
- 속성 탭 클릭 → Next.js Route Handler → Gemini API 스트리밍 호출
- Vercel AI SDK `useChat` / `streamText` 활용
- 해설 섹션 내 관련 이미지(작가 사진, 유사 작품 썸네일) 자동 노출
- "다른 상담하기" 버튼: 동일 속성 재생성

### F4. 톤앤매너 선택
- 해설 하단 또는 우상단 "톤 변경" 버튼으로 진입
- 4가지 카드 선택 UI (배경 이미지 + 설명 예시 문장 포함)
  - 정식적·해설형 → 신뢰성·전문성 강화
  - 유머러스·친근형 → 몰입·긍정감 강화
  - 어린이 맞춤형 → 가족 관람객 접근성
  - 철학적·사색형 → 정서적·회복적 경험
- 선택 즉시 현재 속성 기준 해설 재생성

### F5. 추가 질문 (멀티턴 채팅)
- 해설 하단 고정 채팅 입력창 + 마이크 아이콘
- 이전 해설 + 대화 히스토리를 context로 Gemini 재호출
- 세션 내 대화 유지 (Zustand로 conversation history 관리)

## 4. 수익화 방향 (Monetization)

> 포트폴리오 발표 시 "확장 가능성" 섹션으로 활용 가능

### M1. 기관 B2B SaaS (핵심)
- **대상**: 미술관, 박물관, 갤러리, 문화재단
- **모델**: 월정액 구독 (작품 수 tier 기반)
  - Starter: 작품 50개 이하 / ₩99,000/월
  - Pro: 작품 300개 이하 / ₩299,000/월
  - Enterprise: 무제한 + 커스텀 톤 추가 / 협의
- **제공 가치**: 도슨트 인건비 절감, 24시간 운영, 다국어 확장 가능

### M2. QR코드 연동 현장 서비스
- 작품 옆 QR 스캔 → 해당 작품 상세 페이지 직접 진입
- 기관이 QR 스티커 구매하는 방식 (하드웨어 번들)
- 현장 관람 중 스마트폰으로 즉시 해설 접근

### M3. 프리미엄 톤 팩 (개인 사용자)
- 기본 4가지 톤은 무료
- 추가 톤 DLC 방식으로 판매
  - 예: "시인의 언어형", "영화 나레이터형", "역사학자형"
  - 톤 팩 1개 ₩1,900 or 전체 팩 ₩4,900

### M4. 전시 큐레이션 리포트
- 관람객이 어떤 속성·톤을 많이 선택했는지 데이터 분석 리포트
- 기관 대상 월별 리포트 제공 (B2B 부가 서비스)
- "이번 달 관람객 63%가 '작품 배경'에 관심" 형태의 인사이트

### M5. 교육기관 연계 (장기)
- 학교 미술 수업용 "어린이 맞춤형" 전용 라이선스
- 교사가 작품 세트를 큐레이션해서 학생들에게 링크 공유
- 학교 단위 연간 구독

## 5. 기술 스택

### Frontend
| 역할 | 선택 | 근거 |
|------|------|------|
| 프레임워크 | Next.js 15 (App Router) | SSR로 작품 페이지 SEO, 익숙한 스택 |
| 언어 | TypeScript | 타입 안정성 |
| 스타일 | SCSS Modules | 컴포넌트 스코프, 익숙한 환경 |
| 상태관리 | Zustand | 톤 선택, 대화 히스토리 전역 상태 |
| 서버 상태 | TanStack Query | 작품 목록 캐싱, 무한 스크롤 |
| AI 스트리밍 | Vercel AI SDK (`ai` 패키지) | Gemini 스트리밍 처리 |
| 애니메이션 | Framer Motion | 탭 전환, 해설 페이드인 |

### Backend / Infra
| 역할 | 선택 | 근거 |
|------|------|------|
| DB | Supabase (PostgreSQL) | 익숙한 환경, 작품·작가 데이터 |
| 스토리지 | Supabase Storage | 작품 이미지 |
| AI | Gemini 1.5 Flash | 비용 효율, 한국어 품질 양호 |
| API | Next.js Route Handlers | Gemini API 키 서버사이드 보호 |
| 배포 | Vercel | Next.js 최적화, 무료 티어 |

### 향후 확장 (논문 향후 연구방향 반영)
| 기술 | 용도 |
|------|------|
| LangChain + Self-RAG | 작품 설명 품질 향상, 환각 감소 |
| pgvector (Supabase) | 작품 임베딩 기반 유사 작품 추천 |
| Whisper API | 마이크 음성 입력 STT |

## 8. AI 프롬프트 설계

### 시스템 프롬프트 템플릿
```
당신은 서울시립미술관의 AI 도슨트입니다.
현재 설명 중인 작품:
- 작품명: {title} ({year})
- 작가: {artist_name}
- 기법: {technique} / 재료: {material} / 크기: {dimensions}
- 작가 정보: {artist_bio}

[설명 속성]: {attribute}
[톤앤매너]: {tone}

아래 조건을 반드시 지켜 해설을 작성하세요:
- 분량: 200~300자 (한국어 기준)
- 관람객이 작품 앞에 서 있는 상황을 상상하며 작성
- 마지막 문장은 관람객이 더 궁금해질 만한 여운으로 마무리
- 전문 용어는 쉬운 말로 풀어서 설명
```

### 속성별 추가 지시
| 속성 | 프롬프트 힌트 | Honeycomb 대응 |
|------|-------------|----------------|
| 작품 배경 | 시대적 맥락, 작가 생애, 사조, 종교·사회적 배경 중심 | 유용성, 신뢰성 |
| 작품 의미·상징 | 제목 어원, 색채·구도·소재의 상징, 작가의 메시지 중심 | 매력성, 가치성 |
| 작품 관계성 | 동시대 작가, 유사 작품, 미술사적 위치, 영향 관계 중심 | 탐색성, 가치성 |

### 톤별 문체 지시
| 톤 | 문체 지시 |
|----|----------|
| 정식적·해설형 | 존댓말, 학술적 어휘, 객관적 서술, 신뢰감 있는 어조 |
| 유머러스·친근형 | 반말체 가능, 비유·농담 활용, 가볍고 친근한 호기심 유발 |
| 어린이 맞춤형 | 쉬운 단어, 짧은 문장, 의성어·의태어 활용, 흥미 위주 |
| 철학적·사색형 | 질문형 문장 포함, 내면을 돌아보는 어조, 여백 있는 표현 |
